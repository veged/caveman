# Russian Caveman 🇷🇺🪨

**Зачем много токенов, когда мало хватит.**

Русская версия [Caveman](../../README.md). Сжимает русскоязычные ответы ИИ, сохраняя смысл и технику. Красота вторична.

---

## Что это

Caveman — навык для Claude Code и других ИИ-агентов, который режет воду и «гладкий» стиль в ответах. Хорошо работает для английского.

**Russian Caveman** — адаптация под русский. Русский обычно длиннее английского в токенах:
- длиннее словоформы
- больше служебных слов
- тяжёлые синтаксические конструкции
- склонность ИИ писать «гладко», а не компактно

Русский Caveman:
1. Режет вводные, вежливые обёртки, паразитные конструкции
2. Сжимает синтаксис (двоеточие, тире, стрелка вместо длинных связок)
3. Опускает подлежащее, если оно понятно
4. Оставляет код / URL / API / JSON / имена неприкосновенными
5. Использует только общеизвестные русские сокращения из списка разрешённых

## До и после

### Пример 1 — объяснение ошибки

**Было** (61 слово):
> «Скорее всего, проблема заключается в том, что объект создаётся на каждом рендере, поэтому React считает свойство новым и повторно рендерит компонент. Используйте `useMemo`, чтобы сохранить ссылку между рендерами.»

**ru-lite** (23 слова):
> «Объект создаётся на каждом рендере. React видит новое свойство и делает повторный рендер. Используйте `useMemo`.»

**ru-full** (15 слов):
> «Новый объект каждый рендер → React видит новое свойство → повторный рендер. Оберни в `useMemo`.»

**ru-ultra** (11 слов):
> «Каждый рендер: новый объект → новое свойство → повторный рендер. Решение: `useMemo`.»

**ru-notes** (9 слов):
> «каждый рендер: новый объект → новое свойство → повторный рендер. исправление: `useMemo`.»

### Пример 2 — отладка

**Было:**
> «Скорее всего, проблема заключается в том, что сервис не может подключиться к базе данных, потому что переменная окружения `DATABASE_URL` не задана.»

**ru-full:**
> «Проблема: сервис не подключается к БД — `DATABASE_URL` не задана. Исправление: задать переменную окружения.»

**ru-ultra:**
> «Нет `DATABASE_URL` → нет подключения к БД.»

### Пример 3 — пул соединений

**Было:**
> «Пул соединений позволяет переиспользовать уже открытые соединения с базой данных вместо того, чтобы создавать новое соединение на каждый запрос. Это позволяет избежать накладных расходов на повторное установление соединения.»

**ru-full:**
> «Пул = переиспользование соединений. Нет нового соединения на запрос. Экономит установление связи.»

**ru-ultra:**
> «Пул переиспользует соединения → без установления связи → быстрее под нагрузкой.»

**ru-notes:**
> «пул: переиспользование соединений, без установления связи, быстро под нагрузкой.»

## Уровни

| Уровень     | Команда                                  | Назначение                                       |
|-------------|------------------------------------------|--------------------------------------------------|
| **ru-lite** | `/caveman ru-lite`                       | пользовательские ответы, документация, мало воды  |
| **ru-full** | `/caveman ru-full` или `/caveman ru`     | **по умолчанию**. Технические объяснения, проверка кода, отладка |
| **ru-ultra** | `/caveman ru-ultra`                     | ответы между агентами, внутренние сводки          |
| **ru-notes** | `/caveman ru-notes`                     | конспект, «для себя или другого агента»           |

Альтернативный синтаксис: `/caveman-ru [lite|full|ultra|notes]`.

Уровень держится до смены или конца сессии.

**Выключить:** `stop caveman`, `обычный режим`, `нормальный режим`.

---

## Когда какой уровень

- **ru-lite** — текст для пользователя, документация, публичный README. Важна вежливость.
- **ru-full** — по умолчанию для технических объяснений, проверок кода, сеансов отладки. Короткие фразы, но читается без словаря.
- **ru-ultra** — обмен между агентами, сводки по проекту, когда контекст уже известен.
- **ru-notes** — «заметка для себя через неделю» или «для следующего агента, который сюда придёт».

### Когда НЕ использовать ultra / notes

Автоматический выход из режима:
- Предупреждения о безопасности (SQL-инъекции, XSS, обход авторизации…)
- Подтверждения необратимых операций (`DROP TABLE`, `rm -rf`, `git push --force`)
- Пошаговые инструкции, где неверный порядок всё ломает
- Пользователь запутался, переспрашивает, жалуется на непонятность

После опасной или неясной части — режим возвращается.

## Поведение по типу запроса

### Обычный вопрос
Ответ в выбранном уровне сжатия.

### Неоднозначный вопрос
Если сжатие делает ответ двусмысленным — удлинить. Понятность важнее размера.

### Пошаговая инструкция
Короткие нумерованные шаги. Минимум вводных. Максимум действия.
```text
1. Поставь `foo`
2. Запусти `bar --init`
3. Проверь логи
```

### Сравнение
Таблица, пары `A — …`, `B — …` или формат «плюсы / минусы».

### Ответ с кодом
Прозу — сжать. Код — не трогать. Объяснение — одной фразой до или после блока.

## Структурные шаблоны

Модель предпочитает эти паттерны при сжатии:
- `причина → следствие`
- `симптом → диагноз → исправление`
- `вход → шаги → результат`
- `было / стало`
- `риск / причина / мера`
- `что сломано → почему → что делать`

---

## Инварианты — НЕ ТРОГАТЬ

Никакой режим не сокращает и не переписывает:

- код (блоки ```)
- команды оболочки
- URL, пути, имена файлов
- имена API, функций, классов, переменных, методов
- названия библиотек, фреймворков, продуктов
- JSON / YAML / TOML / XML / SQL
- трассировки стека, строки логов, вывод различий
- цитаты ошибок — дословно

## Список разрешённых сокращений

Разрешены только общеизвестные, однозначные в техническом контексте:

`т.к.`, `т.е.`, `и т.д.`, `и т.п.`, `в т.ч.`, `и др.`, `напр.`, `см.`, `ср.`, `кол-во`, `к-рый`, `св-во`, `гос-во`, `р-н`, `ж/д`, `ЧТД`.

Полный список: [`abbreviations.md`](./abbreviations.md).

**Запрещено:**
- Изобретать новые сокращения «на ходу»
- Сокращать редкие слова до неузнаваемости
- Транслитерировать кодовые сущности (`useMemo` ≠ «юзМемо»)
- Сокращать технические термины, если это ломает поиск или понимание

## Риски

| Риск                                           | Мера                                            |
|------------------------------------------------|-------------------------------------------------|
| Слишком жёсткое сжатие → двусмысленность       | Автоматическая ясность + приоритет понятности   |
| Сокращения ухудшают читаемость                 | Список разрешённых, никакого изобретательства    |
| Смешение русского и английского → странный стиль | Кодовые сущности на английском, остальное на русском |
| Уровень ultra там, где нужен lite              | Таблица уровней + примеры в этом файле          |

## Как выбрать язык

Три способа.

### 1. Команда в каждом запросе

Работает везде, где есть обработчик:

```text
/caveman ru          # ru-full (по умолчанию)
/caveman ru-lite
/caveman ru-full
/caveman ru-ultra
/caveman ru-notes

# или эквивалент
/caveman-ru
/caveman-ru lite
/caveman-ru ultra
/caveman-ru notes

# обратно на английский
/caveman full
```

### 2. Язык по умолчанию через переменную окружения

Claude Code читает `env` из `~/.claude/settings.json` при старте сессии:

```json
{
  "env": {
    "CAVEMAN_LANG": "ru"
  }
}
```

Каждая новая сессия стартует в `ru-full`, обработчик при старте внедряет русские правила. `"en"` или отсутствие — английский по умолчанию.

Или через оболочку:
```bash
export CAVEMAN_LANG=ru
```

### 3. Автоматическое определение по тексту запроса

Включается явно:

```json
{
  "env": {
    "CAVEMAN_AUTO_DETECT_LANG": "1"
  }
}
```

Как работает: обработчик `UserPromptSubmit` смотрит на каждый запрос, игнорируя блоки кода и строчный код. ≥30 % кириллицы → переключает `full` → `ru-full`, `lite` → `ru-lite`, `ultra` → `ru-ultra`. Почти чистая латиница (≤5 % кириллицы) → откатывает. Короткие запросы (менее 15 символов) не обрабатываются.

Явная команда `/caveman …` всегда перебивает автоматическое определение.

### Строка состояния

`[CAVEMAN]`, `[CAVEMAN:ULTRA]`, `[CAVEMAN:RU-FULL]`, `[CAVEMAN:RU-NOTES]` — показывает активный режим и язык.

### Для других агентов

CI синхронизирует русские файлы правил вместе с английскими:

| Агент    | Файл                               | Поведение                                                      |
|----------|-------------------------------------|----------------------------------------------------------------|
| Cline    | `.clinerules/caveman-ru.md`         | Загружается автоматически вместе с английским. Модель решает.  |
| Cursor   | `.cursor/rules/caveman-ru.mdc`      | `alwaysApply: false` — по запросу агента                       |
| Windsurf | `.windsurf/rules/caveman-ru.md`     | `trigger: model_decision`                                      |
| Copilot  | —                                    | Не внедряется. Общая инструкция репозитория остаётся английской |

Для Copilot: `CAVEMAN_LANG=ru` в переменных окружения или ссылка на `skills/caveman-ru/SKILL.md`.
