# Russian Caveman 🇷🇺🪨

**Зачем много токен, когда мало хватит.**

Русская версия [Caveman](../../README.md). Сжимает русскоязычные ответы ИИ, оставляя смысл и технику. Красота — вторична.

---

## Что это

Caveman — скилл для Claude Code и других AI-агентов, который режет воду и «гладкий» стиль в ответах. Работает хорошо для английского.

**Russian Caveman** — адаптация под русский. Русский обычно длиннее английского в токенах:
- длиннее словоформы;
- больше служебных слов;
- тяжёлые синтаксические конструкции;
- склонность ИИ писать «гладко», а не компактно.

Русский caveman делает:
1. режет вводные, вежливые обёртки, паразиты;
2. сжимает синтаксис (двоеточие, тире, стрелка → вместо длинных связок);
3. опускает подлежащее, если понятно;
4. оставляет код / URL / API / JSON / имена неприкосновенными;
5. использует только общеизвестные русские сокращения из whitelist.

## Before / After

### Пример 1 — объяснение бага

**Было** (61 слово):
> «Скорее всего, проблема заключается в том, что объект создаётся на каждом рендере, поэтому React считает prop новым и повторно рендерит компонент. Используйте `useMemo`, чтобы сохранить ссылку между рендерами.»

**ru-lite** (23 слова):
> «Объект создаётся на каждом рендере. React видит новый prop и делает повторный рендер. Используйте `useMemo`.»

**ru-full** (15 слов):
> «Новый объект каждый рендер → React видит новый prop → повторный рендер. Оберни в `useMemo`.»

**ru-ultra** (11 слов):
> «Каждый рендер: новый object → новый prop → re-render. Решение: `useMemo`.»

**ru-notes** (9 слов):
> «new object each render → prop changed → re-render. fix: `useMemo`.»

### Пример 2 — debug

**Было:**
> «Скорее всего, проблема заключается в том, что сервис не может подключиться к базе данных, потому что переменная окружения `DATABASE_URL` не задана.»

**ru-full:**
> «Проблема: сервис не подключается к БД — `DATABASE_URL` не задана. Фикс: задать env.»

**ru-ultra:**
> «Нет `DATABASE_URL` → нет коннекта к БД. Фикс: set env.»

## Уровни

| Уровень | Команда | Назначение |
|---------|---------|-----------|
| **ru-lite** | `/caveman ru-lite` | user-facing, документация, минимум воды |
| **ru-full** | `/caveman ru-full` или `/caveman ru` | **default**. Техобъяснения, review, debug |
| **ru-ultra** | `/caveman ru-ultra` | Agent-to-agent, внутренние summaries |
| **ru-notes** | `/caveman ru-notes` | Конспект, «для себя или другого агента» |

Альтернативный синтаксис: `/caveman-ru [lite|full|ultra|notes]`.

Уровень держится до смены или конца сессии.

**Выключить:** `stop caveman`, `обычный режим`, `нормальный режим`.

---

## Инварианты — НЕ ТРОГАТЬ

Никакой режим не сокращает и не переписывает:

- код (блоки ```);
- shell-команды;
- URL, пути, имена файлов;
- имена API, функций, классов, переменных, методов;
- названия библиотек, фреймворков, продуктов;
- JSON / YAML / TOML / XML / SQL;
- stack traces, лог-строки, diff;
- цитаты ошибок — дословно.

---

## Whitelist сокращений

Разрешены только общеизвестные, однозначные в техконтексте:

`т.к.`, `т.е.`, `и т.д.`, `и т.п.`, `в т.ч.`, `и др.`, `напр.`, `см.`, `ср.`, `кол-во`, `к-рый`, `св-во`, `гос-во`, `р-н`, `ж/д`, `ЧТД`.

Полный список: [`abbreviations.md`](./abbreviations.md).

**Запрещено:**
- изобретать новые сокращения «на ходу»;
- сокращать редкие слова до неузнаваемости;
- «русифицировать» кодовые сущности (`useMemo` ≠ «юзМемо»);
- сокращать техтермины, если это ломает поиск или понимание.

## Когда какой уровень

- **ru-lite** — пишешь пользователю, в документацию, в публичный README. Важна вежливость.
- **ru-full** — дефолт для техобъяснений, ревью, debug-сессий. Короткие фразы, но читается без словаря.
- **ru-ultra** — agent-to-agent, сводки по проекту, когда контекст уже известен.
- **ru-notes** — «заметка для следующего меня через неделю» или «для следующего агента, который сюда придёт».

## Когда НЕ использовать ultra / notes

Авто-выход из режима (Auto-Clarity):
- security warnings (SQL injection, XSS, auth обход…);
- подтверждения необратимых операций (`DROP TABLE`, `rm -rf`, `git push --force`);
- пошаговые инструкции, где неверный порядок всё ломает;
- пользователь запутался, переспрашивает, жалуется на непонятность.

После опасной / неясной части — режим возвращается.

## Риски

| Риск | Мера |
|------|------|
| Слишком жёсткое сжатие → двусмысленность | Auto-Clarity + приоритет понятности |
| Сокращения ухудшают читаемость | Whitelist, никакого изобретательства |
| RU + EN mix → странный стиль | Кодовые сущности остаются англ., остальное — русское |
| Ultra там, где нужен lite | Таблица уровней + примеры в этом файле |

## Установка

Работает автоматически в Claude Code / Codex, если установлен основной плагин Caveman. Для других агентов — как основной Caveman (см. [основной README](../../README.md#install)).

Активация в сессии:
```
/caveman ru          # ru-full (default)
/caveman ru-lite
/caveman ru-full
/caveman ru-ultra
/caveman ru-notes
```

Или эквивалент:
```
/caveman-ru
/caveman-ru lite
/caveman-ru ultra
/caveman-ru notes
```
