# Russian Caveman — benchmark corpus

Ручной корпус для проверки поведения русского режима. Каждый кейс — пара `baseline` (нормальный русский) + четыре уровня сжатия: `ru-lite`, `ru-full`, `ru-ultra`, `ru-notes`.

Корпус используется:
1. Как **референс** для ревью — проверка, что режим ведёт себя как задумано.
2. Как **regression snapshot** — если поведение меняется, обновляется корпус.
3. Как **training set** для промптов — примеры того, что считается правильным ответом.

## Типы кейсов

Покрываются:
1. объяснение багов;
2. code review comments;
3. shell / debug инструкции;
4. сравнение инструментов;
5. краткие summaries;
6. ответы с кодом;
7. ответы с JSON / YAML;
8. пользовательские ответы;
9. внутренние агентные заметки.

## Оценка

Для каждого кейса — качественная оценка:
- **длина** (символы + приблизительные токены по правилу 3.5 char/token для кириллицы);
- **понятность** — понятен ли смысл с первого чтения;
- **сохранность фактов** — не потеряны ли причинно-следственные связи;
- **сохранность формальных сущностей** — код/URL/имена не искажены.

Файл: [`examples.json`](./examples.json) — машиночитаемая таблица примеров.

## Инварианты (обязательно для всех уровней)

- Код, shell-команды, пути, URL, API, JSON/YAML/SQL, stack traces — не переписывать и не сокращать.
- Имена функций, классов, библиотек, переменных — как есть.
- Цитаты ошибок — дословно.
