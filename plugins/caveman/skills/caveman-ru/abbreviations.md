# Russian Caveman — whitelist сокращений

Только эти сокращения разрешены в ответах режима `caveman-ru`. Всё остальное — либо полным словом, либо оставлять как есть (для кодовых сущностей).

Принципы:
1. **Общеизвестно** — понятно любому носителю русского.
2. **Однозначно** — в техконтексте не порождает двусмысленности.
3. **Устойчиво** — закреплено в письменной практике, а не изобретено.

## Базовый whitelist

### Служебные
| Сокр. | Расшифровка |
|-------|------------|
| `т.к.` | так как |
| `т.е.` | то есть |
| `и т.д.` | и так далее |
| `и т.п.` | и тому подобное |
| `и др.` | и другие |
| `в т.ч.` | в том числе |
| `напр.` | например |
| `см.` | смотри |
| `ср.` | сравни |
| `п.` | пункт |
| `пп.` | пункты |
| `гл.` | глава |
| `рис.` | рисунок |
| `табл.` | таблица |
| `стр.` | страница |
| `прим.` | примечание |

### Графические
| Сокр. | Расшифровка |
|-------|------------|
| `кол-во` | количество |
| `к-рый` | который |
| `св-во` | свойство |
| `гос-во` | государство |
| `стр-во` | строительство |
| `р-н` | район |
| `ж/д` | железнодорожный / железная дорога |
| `п/о` | почтовое отделение |

### Математические / логические
| Сокр. | Расшифровка |
|-------|------------|
| `ЧТД` | что и требовалось доказать |
| `≈` | приблизительно |
| `≠` | не равно |
| `⇒` / `→` | следовательно / приводит к |
| `⇔` | тогда и только тогда |

## Tech-контекст (оставлять как есть)

Кодовые сущности, аббревиатуры индустрии — не переводить и не переписывать кириллицей. Примеры того, что **остаётся в исходном виде**:

- `DB`, `SQL`, `API`, `HTTP`, `TCP`, `UDP`, `JSON`, `YAML`, `TOML`, `XML`, `CSV`
- `auth`, `config`, `req`, `res`, `fn`, `impl`, `conn`, `pool`, `env`, `var`, `ref`, `prop`, `props`
- `CI`, `CD`, `CI/CD`, `PR`, `MR`, `repo`, `diff`, `HEAD`, `origin`, `upstream`
- `OS`, `CPU`, `GPU`, `RAM`, `IO`, `FS`, `TLS`, `SSL`, `DNS`, `CDN`
- `RPS`, `QPS`, `TPS`, `P99`, `RTT`, `SLA`, `SLO`, `SLI`
- `K8s`, `k8s`, `Docker`, `npm`, `pip`, `cargo`, `go mod`
- `useMemo`, `useState`, `useEffect`, `async`, `await`, `Promise` — идентификаторы, не переводить

## Строго запрещено

- ❌ Изобретать новые сокращения «на ходу» (например, `прбл.`, `фнкц.`, `объкт` — такое недопустимо).
- ❌ Сокращать редкие слова до неузнаваемости.
- ❌ «Русифицировать» кодовые сущности: `useMemo` → ~~«юзМемо»~~.
- ❌ Сокращать ключевые технические термины, если это ломает полнотекстовый поиск или понимание для новичка.
- ❌ Смешивать сокращения разных доменов так, что контекст теряется.

## Расширение словаря

Если понадобилось новое сокращение — не добавлять его в ответ. Предпочесть полное слово. Расширение этого файла делается только отдельным коммитом с обоснованием и тестом на неоднозначность.
